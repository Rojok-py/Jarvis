# J.A.R.V.I.S. — Voice & Text AI Assistant

> **Just A Rather Very Intelligent System**
>
> Модульный AI-ассистент для Arch Linux с голосовым и текстовым режимами на базе Groq (Llama 3.3 70B).
> Поддерживает RAG (поиск по документам), веб-поиск, анализ файлов и голосовое управление.

---

## Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.12+ |
| Пакетный менеджер | Poetry |
| LLM | Llama 3.3 70B через Groq API (`groq`) |
| Embeddings | TF-IDF (`scikit-learn`, локально) |
| STT | Whisper Large V3 Turbo через Groq API |
| TTS | `edge-tts` (Microsoft Azure voices) + `mpv` |
| Запись звука | FFmpeg через `subprocess` (PulseAudio, без PyAudio/PortAudio) |
| RAG | PyMuPDF + scikit-learn TF-IDF + NumPy (косинусное сходство) |
| UI | PyQt6, тёмная футуристическая тема + фоновое изображение |
| Веб-поиск | DuckDuckGo (`duckduckgo-search`) |
| ОС | Arch Linux |

## Установка (пошагово)

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Rojok-py/Jarvis.git
cd Jarvis
```

### 2. Установить системные зависимости (Arch Linux)

```bash
sudo pacman -S ffmpeg mpv python python-pip
```

> FFmpeg — для записи с микрофона, mpv — для воспроизведения озвучки.

### 3. Установить Poetry (если ещё нет)

```bash
curl -sSL https://install.python-poetry.org | python3 -
```

Убедитесь, что Poetry в PATH:
```bash
poetry --version
```

### 4. Установить Python-зависимости

```bash
poetry install
```

Это создаст виртуальное окружение `.venv/` и установит все пакеты:
- `groq` — Groq SDK (Llama 3.3 + Whisper)
- `scikit-learn` — TF-IDF эмбеддинги для RAG
- `edge-tts` — синтез речи
- `PyQt6` — графический интерфейс
- `python-dotenv` — чтение `.env`
- `duckduckgo-search` — веб-поиск
- `pymupdf` — парсинг PDF для RAG
- `numpy` — вычисление косинусного сходства

### 5. Получить API-ключ Groq

1. Перейдите на [Groq Console](https://console.groq.com/keys)
2. Создайте API-ключ
3. Создайте файл `.env` в корне проекта:

```bash
echo "GROQ_API_KEY=ваш_ключ" > .env
```

### 6. (Опционально) Добавить файлы для RAG

Положите PDF, TXT, MD, CSV, JSON или PY файлы в `data/`:

```bash
cp ~/Documents/report.pdf data/
```

## Запуск

```bash
poetry run python src/main.py
```

После запуска откроется окно J.A.R.V.I.S. в текстовом режиме.

## Использование

### Текстовый режим
Введите сообщение в поле ввода и нажмите Enter или кнопку «Отправить».

### Голосовой режим
Нажмите кнопку «Режим: ТЕКСТ» → переключится на «Режим: ГОЛОС».
Нажмите 🎤 для записи (5 секунд) → распознавание → ответ + озвучка.

### Голосовые / текстовые команды
| Команда | Действие |
|---------|----------|
| `проанализируй файл report.pdf` | Анализ файла через Groq |
| `проанализируй файл report.pdf и сохрани в summary.txt` | Анализ + сохранение в out/ |
| `найди в интернете ...` / `поищи ...` | Веб-поиск DuckDuckGo + Groq |
| `покажи файлы` / `список файлов` | Листинг data/ и out/ |
| `сброс чата` / `новый чат` | Очистка истории диалога |
| `индексируй файлы` / `обнови индекс` | Индексация файлов для RAG |
| `по документам ...` / `по файлам ...` | RAG-запрос по проиндексированным файлам |

### RAG (поиск по документам)
1. Положите файлы в `data/` (PDF, TXT, MD и др.)
2. Нажмите кнопку **📑 Индексация** в интерфейсе (или скажите «индексируй файлы»)
3. Задавайте вопросы: `по документам что такое алгоритм?`

RAG автоматически:
- Извлекает текст из PDF (PyMuPDF) и текстовых файлов
- Разбивает на чанки (800 символов с перекрытием 200)
- Строит TF-IDF векторы локально через scikit-learn
- Кеширует индекс в `.rag_cache/` (не пересчитывает если файл не изменился)
- При запросе ищет топ-5 релевантных фрагментов по косинусному сходству
- Передаёт найденный контекст в Groq для формирования ответа

## Что реализовано

### `src/core/engine.py` — Groq Engine
- **`send_text()`** — текстовый чат с историей (Groq Chat Completions)
- **`send_audio()`** — транскрипция через Whisper + ответ через чат
- **`analyze_file()`** — чтение файла + анализ через Groq
- **`rag_query()`** — запрос с RAG-контекстом из проиндексированных документов
- **`index_documents()`** — индексация файлов из data/
- **`process_voice_command()`** — маршрутизация команд по интентам
- **`reset_chat()`** — сброс истории диалога

### `src/core/rag.py` — RAG-модуль
- **`RAGIndex`** — класс для индексации и семантического поиска
- Парсинг PDF через PyMuPDF, текстовых файлов напрямую
- Чанкинг с перекрытием (800/200 символов)
- TF-IDF векторизация через scikit-learn (локально, без GPU)
- Кеширование индекса на диск (JSON + .npy)
- Инкрементальная индексация (по хешу файла)
- Косинусное сходство через NumPy

### `src/ui/interface.py` — Главное окно
- **Два режима работы**: текстовый и голосовой
- **Фоновое изображение** из `assets/jarvis.webp` (полупрозрачное)
- **GIF-анимация** из `assets/jarvis.gif`
- Кнопка **📑 Индексация** для RAG
- Фоновые потоки: `TextWorker`, `VoiceWorker`, `IndexWorker`
- Футуристическая тёмная тема

### `src/audio/recorder.py` — Запись звука
- FFmpeg + PulseAudio, моно 16 kHz, 16-bit PCM → `data/input.wav`

### `src/audio/speaker.py` — Озвучка
- `edge-tts` (голос `en-GB-RyanNeural`) + `mpv`

### `src/tools/search.py` — Веб-поиск
- DuckDuckGo без API-ключа

### `src/tools/file_ops.py` — Файловые операции
- Листинг, чтение, сохранение файлов
