# J.A.R.V.I.S. — Voice & Text AI Assistant

> **Just A Rather Very Intelligent System**
>
> Модульный AI-ассистент для Arch Linux с голосовым и текстовым режимами на базе Groq (Llama 3.3 70B).
> Поддерживает RAG (поиск по документам), веб-поиск, двухагентный LangChain-пайплайн,
> запись данных в файлы и голосовое управление.

---

## Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.12+ |
| Пакетный менеджер | Poetry |
| LLM | Llama 3.3 70B через Groq API (`groq`) |
| Embeddings | TF-IDF (`scikit-learn`, локально) |
| STT | Whisper Large V3 Turbo через Groq API |
| TTS | `edge-tts` (Microsoft Azure voices) + `mpv` |
| Запись звука | FFmpeg через `subprocess` (PulseAudio, без PyAudio/PortAudio) |
| RAG | PyMuPDF + scikit-learn TF-IDF + NumPy (косинусное сходство) |
| Агенты | LangChain (Research Agent + Editor Agent) + DuckDuckGo |
| UI | PyQt6, тёмная футуристическая тема + фоновое изображение |
| Веб-поиск | DuckDuckGo (`duckduckgo-search`, `langchain-community`) |
| ОС | Arch Linux |

## Установка (пошагово)

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Rojok-py/Jarvis.git
cd Jarvis
```

### 2. Установить системные зависимости (Arch Linux)

```bash
sudo pacman -S ffmpeg mpv python python-pip
```

> FFmpeg — для записи с микрофона, mpv — для воспроизведения озвучки.

### 3. Установить Poetry (если ещё нет)

```bash
curl -sSL https://install.python-poetry.org | python3 -
```

Убедитесь, что Poetry в PATH:
```bash
poetry --version
```

### 4. Установить Python-зависимости

```bash
poetry install
```

Это создаст виртуальное окружение `.venv/` и установит все пакеты:
- `groq` — Groq SDK (Llama 3.3 + Whisper)
- `scikit-learn` — TF-IDF эмбеддинги для RAG
- `edge-tts` — синтез речи
- `PyQt6` — графический интерфейс
- `python-dotenv` — чтение `.env`
- `duckduckgo-search` — веб-поиск
- `pymupdf` — парсинг PDF для RAG
- `numpy` — вычисление косинусного сходства
- `langchain` + `langchain-community` + `langchain-openai` — двухагентный пайплайн

### 5. Получить API-ключ Groq

1. Перейдите на [Groq Console](https://console.groq.com/keys)
2. Создайте API-ключ
3. Создайте файл `.env` в корне проекта:

```bash
echo "GROQ_API_KEY=ваш_ключ" > .env
```

### 6. (Опционально) Добавить файлы для RAG

Положите PDF, TXT, MD, CSV, JSON или PY файлы в `data/`:

```bash
cp ~/Documents/report.pdf data/
```

## Использование

### Текстовый режим
Введите сообщение в поле ввода и нажмите Enter или кнопку «Отправить».

### Голосовой режим
Нажмите кнопку «Режим: ТЕКСТ» → переключится на «Режим: ГОЛОС».
Нажмите 🎤 для записи (5 секунд) → распознавание → ответ + озвучка.

### Команды (текстовые и голосовые)

| Команда | Действие |
|---------|----------|
| `проанализируй файл report.pdf` | Анализ файла через Groq |
| `проанализируй файл report.pdf и сохрани в summary.txt` | Анализ + сохранение в out/ |
| `что в моём файле написано про X` | RAG-запрос по файлам (естественный язык) |
| `расскажи про алгоритмы в моём файле` | RAG-запрос — авто-индексация + поиск |
| `прочитай файл` | RAG-запрос с автоматическим поиском по документам |
| `запиши в файл notes.txt: текст...` | Создать/перезаписать файл с содержимым |
| `сохрани в файл results.txt: данные` | Аналогично — создание/перезапись файла |
| `найди в интернете ...` / `поищи ...` | Веб-поиск DuckDuckGo + Groq |
| `запусти агентов про тему X` | Двухагентный LangChain-пайплайн (Research → Editor) |
| `исследуй с агентами тему X` | Запуск Research + Editor агентов |
| `покажи файлы` / `список файлов` | Листинг data/ и out/ |
| `сброс чата` / `новый чат` | Очистка истории диалога |
| `индексируй файлы` / `обнови индекс` | Индексация файлов для RAG |
| `по документам ...` / `по файлам ...` | Явный RAG-запрос по проиндексированным файлам |
| `создай файл notes.txt ...` | Создать текстовый файл (LLM генерирует содержимое) |
| `измени файл notes.txt ...` | Отредактировать существующий файл через LLM |
| `допиши в файл notes.txt ...` | Добавить текст в конец файла |

## Двухагентный LangChain-пайплайн

J.A.R.V.I.S. включает систему из двух взаимодействующих агентов на базе **LangChain**:

```
Пользователь
     │
     ▼
┌─────────────────────────────────────────────┐
│           Research Agent                     │
│  • Анализирует запрос пользователя           │
│  • Определяет, нужен ли веб-поиск           │
│  • Ищет данные через DuckDuckGo (авто)      │
│  • Составляет подробный черновик ответа      │
└───────────────────┬─────────────────────────┘
                    │ черновик
                    ▼
┌─────────────────────────────────────────────┐
│           Editor Agent (J.A.R.V.I.S.)        │
│  • Проверяет логику и достоверность фактов   │
│  • Убирает лишнее, оставляет суть           │
│  • Форматирует в фирменном стиле Jarvis     │
│  • Добавляет вежливость и технологичность   │
└───────────────────┬─────────────────────────┘
                    │ финальный ответ
                    ▼
              Пользователь
```

**Автоматический поиск:** если запрос содержит слова «новости», «погода», «сегодня»,
«актуально», «текущий», «последнее» и т.д. — Research Agent автоматически запускает
DuckDuckGo-поиск перед формированием черновика.

**Примеры использования:**
```
запусти агентов про квантовые компьютеры
исследуй с агентами тему GPT-4
через агентов расскажи про погоду в Москве
dual agent — последние новости по AI
```

## Запись текста в файлы

J.A.R.V.I.S. умеет записывать текст в файлы по голосовой или текстовой просьбе:

**Создание/перезапись файла:**
```
запиши в файл notes.txt: Встреча в пятницу в 15:00
сохрани в файл todo.md: список задач на эту неделю
```

**Создание файла с LLM-генерацией содержимого:**
```
создай файл plan.txt с планом проекта на 3 месяца
напиши файл readme.md с описанием Python-проекта
```

**Дополнение существующего файла:**
```
допиши в файл notes.txt: также не забыть купить молоко
добавь в файл tasks.txt: завершить отчёт
```

**Редактирование через LLM:**
```
измени файл notes.txt и добавь заголовок
отредактируй файл plan.txt — перепиши второй пункт
```

## RAG (поиск по документам)

1. Положите файлы в `data/` (PDF, TXT, MD и др.)
2. Нажмите кнопку **📑 Индексация** (или скажите «индексируй файлы»)
3. Задавайте вопросы в любой форме:

```
по документам что такое алгоритм?
расскажи про алгоритмы в моём файле
что написано в моих файлах про сортировку?
прочитай файл и расскажи ключевые идеи
в моём файле есть информация про нейросети?
```

**Поддерживаемые форматы:** PDF, TXT, MD, CSV, JSON, PY.

RAG автоматически:
- Извлекает текст из PDF (PyMuPDF) и текстовых файлов
- Разбивает на чанки (800 символов с перекрытием 200)
- Строит TF-IDF векторы локально через scikit-learn
- Кеширует индекс в `.rag_cache/`
- При запросе ищет топ-5 релевантных фрагментов по косинусному сходству
- Передаёт найденный контекст в Groq для формирования ответа

## Что реализовано

### `src/core/engine.py` — Groq Engine
- **`send_text()`** — текстовый чат с историей (Groq Chat Completions)
- **`send_audio()`** — транскрипция через Whisper + ответ через чат
- **`analyze_file()`** — чтение файла + анализ через Groq
- **`rag_query()`** — запрос с RAG-контекстом из проиндексированных документов
- **`index_documents()`** — индексация файлов из data/
- **`process_voice_command()`** — маршрутизация команд (улучшена, 9 интентов)
- **`_handle_create_file()`** — создание файла (LLM генерирует содержимое)
- **`_handle_write_to_file()`** — явная запись текста в файл
- **`_handle_edit_file()`** — редактирование файла по инструкции
- **`_handle_append_file()`** — дополнение файла
- **`_handle_dual_agent()`** — запуск LangChain двухагентного пайплайна
- **`reset_chat()`** — сброс истории диалога

### `src/core/agents.py` — LangChain Dual-Agent
- **`run_dual_agent()`** — Research Agent → Editor Agent пайплайн (LCEL)
- **`run_dual_agent_with_search()`** — принудительный запуск с DuckDuckGo
- Автоматическое обнаружение запросов, требующих актуальных данных
- Интеграция DuckDuckGo через `langchain-community`
- Groq через OpenAI-совместимый endpoint (`langchain-openai`)

### `src/core/rag.py` — RAG-модуль
- Парсинг PDF (PyMuPDF), текстовых файлов напрямую
- Чанкинг с перекрытием (800/200 символов)
- TF-IDF векторизация (scikit-learn, локально, без GPU)
- Кеширование и инкрементальная индексация по SHA-256 хешу

### `src/ui/interface.py` — Главное окно
- Текстовый и голосовой режимы
- `TextWorker` вызывает `process_voice_command` для обоих режимов
- Футуристическая тёмная тема PyQt6

### `src/audio/` — Аудио
- `recorder.py` — FFmpeg + PulseAudio → WAV 16 kHz моно
- `speaker.py` — edge-tts + mpv, очистка markdown перед озвучкой

### `src/tools/` — Утилиты
- `search.py` — DuckDuckGo без API-ключа
- `file_ops.py` — листинг, чтение, создание, редактирование файлов

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Jarvis говорит «нет доступа к файлам» | Убедитесь что файлы лежат в `data/`, нажмите «📑 Индексация» |
| Запрос «в моём файле» не распознаётся | Используйте: «в моём файле», «прочитай файл», «по документам» |
| Агенты не запускаются | Проверьте `GROQ_API_KEY` в `.env` |
| Ошибка записи в файл | Убедитесь что папка `data/` существует (создаётся автоматически) |
| 429 Rate Limit | Встроенный retry с паузой 5–15 секунд |
| `invalid style override 'kvantum'` | Косметическая ошибка Qt, не влияет на работу |
